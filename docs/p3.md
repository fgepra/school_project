## 개요

5~6주차는 서비스를 **상용 수준에 가깝게 완성**하는 단계였다.  
결제 플로우(카드 정보 입력 → 처리 → 완료), 강사 정산, DB 기반 알림 시스템,  
서버 활동 로그, cron 기반 스케줄러, 실시간 서버 모니터링을 구현했다.  
또한 Turbopack 환경에서 발생하던 MediaPipe CJS 번들링 오류를 CDN 방식으로 완전 해결했다.

## 1. DB 스키마 설계

### 신규 테이블 4개

#### PAYMENTS — 결제 내역

```sql
CREATE TABLE IF NOT EXISTS payments (
  id             INT AUTO_INCREMENT PRIMARY KEY,
  user_id        INT NOT NULL,
  course_id      INT NOT NULL,
  amount         INT NOT NULL,
  status         ENUM('pending','completed','refunded') DEFAULT 'completed',
  payment_method VARCHAR(50),  -- 카드사명
  card_last4     CHAR(4),      -- 카드 끝 4자리
  created_at     DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id)   REFERENCES users(id)   ON DELETE CASCADE,
  FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE
);
```

#### SETTLEMENTS — 강사 정산

```sql
CREATE TABLE IF NOT EXISTS settlements (
  id            INT AUTO_INCREMENT PRIMARY KEY,
  instructor_id INT NOT NULL,
  course_id     INT NOT NULL,
  period        VARCHAR(7) NOT NULL,       -- 'YYYY-MM' 형식
  revenue       INT DEFAULT 0,             -- 해당 월 총 판매액
  payout_rate   DECIMAL(3,2) DEFAULT 0.70, -- 정산 비율 (기본 70%)
  payout_amount INT DEFAULT 0,             -- 실 지급액
  status        ENUM('pending','paid') DEFAULT 'pending',
  created_at    DATETIME DEFAULT NOW(),
  FOREIGN KEY (instructor_id) REFERENCES users(id),
  FOREIGN KEY (course_id)     REFERENCES courses(id)
);
```

#### NOTIFICATIONS — 알림

```sql
CREATE TABLE IF NOT EXISTS notifications (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  user_id    INT NOT NULL,
  type       VARCHAR(50) NOT NULL,    -- 'payment','course_update','progress' 등
  title      VARCHAR(200) NOT NULL,
  message    TEXT,
  is_read    TINYINT(1) DEFAULT 0,
  related_id INT,                     -- 관련 강의 ID 등
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

#### ACTIVITY_LOGS — 서버 활동 로그

```sql
CREATE TABLE IF NOT EXISTS activity_logs (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  user_id    INT,                      -- NULL 가능 (비인증 요청)
  action     VARCHAR(100) NOT NULL,    -- 'LOGIN', 'PURCHASE', 'VIEW_COURSE' 등
  detail     TEXT,
  ip_address VARCHAR(45),
  created_at DATETIME DEFAULT NOW()
);
```

#### COURSES 테이블 컬럼 추가

```sql
ALTER TABLE courses ADD COLUMN IF NOT EXISTS price INT DEFAULT 0;
```

- `price = 0` 또는 `NULL`이면 무료 강의로 처리.

## 2. 백엔드 구현

### 2-1. 결제 (`backend/controllers/paymentController.js`)

실제 PG(결제대행사) 연동 없이 **시뮬레이션 방식**으로 구현했다.  
카드 정보는 서버에 저장하지 않고, 카드 끝 4자리만 저장한다.

| 메서드 | 경로 | 권한 | 설명 |
|--------|------|------|------|
| POST | `/api/payments` | 학생 | 결제 처리 |
| GET | `/api/payments/my` | 학생 | 내 결제 내역 |
| GET | `/api/payments/:courseId/check` | 학생 | 구매 여부 확인 |
| POST | `/api/payments/:paymentId/refund` | 학생 | 환불 처리 |
| GET | `/api/admin/payments` | 관리자 | 전체 결제 내역 |

**결제 처리 로직**:
```js
// 1. 이미 구매했는지 확인
const existing = await checkPurchase(userId, courseId);
if (existing) return res.status(409).json({ message: '이미 구매한 강의입니다.' });

// 2. 강의 가격 조회 및 검증
const course = await getCourse(courseId);
if (course.price !== amount) return res.status(400).json({ message: '결제 금액 불일치' });

// 3. payments 테이블에 저장
INSERT INTO payments (user_id, course_id, amount, status, payment_method, card_last4)
VALUES (?, ?, ?, 'completed', ?, ?);

// 4. 결제 완료 알림 생성
createNotification(userId, 'payment', '결제 완료', `${course.title} 강의가 결제되었습니다.`);
```

**환불 처리**: `status = 'refunded'`로 업데이트, 실제 금액 환급 없음 (시뮬레이션).

**정산 생성 로직** (월별 cron 또는 수동):
```sql
-- 특정 월의 강의별 매출 집계
SELECT p.course_id, c.instructor_id,
       SUM(p.amount) AS revenue
FROM payments p
JOIN courses c ON p.course_id = c.id
WHERE DATE_FORMAT(p.created_at, '%Y-%m') = ?
  AND p.status = 'completed'
GROUP BY p.course_id, c.instructor_id;
```
집계 후 `payout_amount = revenue × 0.70`으로 계산해 settlements에 INSERT.

### 2-4. 활동 로그 미들웨어 (`backend/middleware/logMiddleware.js`)

모든 인증된 요청에 대해 `activity_logs`에 자동 기록하는 미들웨어.

```js
const logMiddleware = (req, res, next) => {
  const userId = req.user?.id ?? null;
  const action = `${req.method} ${req.path}`;
  const ip = req.ip || req.headers['x-forwarded-for'];
  db.query(
    'INSERT INTO activity_logs (user_id, action, ip_address) VALUES (?,?,?)',
    [userId, action, ip]
  );
  next(); // 비동기 저장, 응답 지연 없음
};
```

### 2-6. 모니터링 미들웨어 (`backend/middleware/metricsMiddleware.js`)

인메모리(메모리 내)에 API 요청 통계를 누적하는 미들웨어.

```js
const metrics = {
  totalRequests: 0,
  errorCount: 0,
  responseTimes: [],   // 최근 100건
  recentRequests: [],  // 최근 50건 상세
};

const metricsMiddleware = (req, res, next) => {
  const start = Date.now();
  metrics.totalRequests++;
  res.on('finish', () => {
    const duration = Date.now() - start;
    metrics.responseTimes.push(duration);
    if (res.statusCode >= 400) metrics.errorCount++;
    metrics.recentRequests.unshift({ method, path, status, duration, timestamp });
    if (metrics.recentRequests.length > 50) metrics.recentRequests.pop();
  });
  next();
};
```

### 2-8. 스케줄러 (`backend/scheduler.js`)

`node-cron`으로 3가지 자동화 작업을 등록했다.

```js
// 1. 매일 새벽 2시 — 30일 이상 오래된 로그 자동 삭제
cron.schedule('0 2 * * *', () => {
  db.query('DELETE FROM activity_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY)');
});

// 2. 매월 1일 자정 — 전월 정산 자동 생성
cron.schedule('0 0 1 * *', () => {
  const lastMonth = dayjs().subtract(1, 'month').format('YYYY-MM');
  generateSettlements(lastMonth);
});

// 3. 매주 월요일 오전 9시 — 진도 50% 미만 학생에게 독려 알림
cron.schedule('0 9 * * 1', () => {
  // progress < 50% 학생 조회 후 notifications INSERT
  sendProgressReminders();
});
```

### 3-2. 결제 페이지 (`/payment/[courseId]`)

실제 카드 결제 UI를 재현한 3단계 플로우.

**Step 1 — 카드 정보 입력**:
- 카드사 선택 (신한, 국민, 현대, 삼성, 롯데, 하나 — 각 고유 색상 블록)
- 카드 번호 자동 포맷팅: `1234-5678-9012-3456` (4자리마다 `-` 자동 삽입)
- 유효기간 `MM/YY` 포맷팅, CVC 3자리
- 할부 선택 (일시불 / 2~12개월)

**Step 2 — 결제 확인**:
- 입력 정보 요약 (강의명, 금액, 카드사, 카드번호 마스킹)
- [결제하기] 버튼 클릭 시 Step 3 진행

**Step 3 — 처리 중**:
- 스피너 + "결제 처리 중..." 메시지 표시
- 1.5초 후 `/payment/complete` 로 이동

```ts
// 결제 시뮬레이션

await new Promise(resolve => setTimeout(resolve, 1500));
await paymentApi.purchase(courseId, { amount: course.price, paymentMethod: selectedCard, cardLast4 });
```

### 3-12. MediaPipe CDN 리팩토링

**문제**: Turbopack이 `@mediapipe/pose`, `@mediapipe/camera_utils`, `@mediapipe/drawing_utils`의 CJS 포맷을 번들링하지 못해 "Cannot find module" 오류 발생.

**해결책**: `await import('@mediapipe/...')` 대신 `<script>` 태그로 CDN에서 동적 로드.

**적용 파일**:
- `hometraining/src/app/(main)/motion/page.tsx`
- `hometraining/src/app/(main)/courses/[id]/lectures/[lectureId]/page.tsx`

```ts
const loadMediaPipe = (): Promise<void> =>
  new Promise((resolve) => {
    if ((window as any).Pose) { resolve(); return; }
    const srcs = [
      'https://cdn.jsdelivr.net/npm/@mediapipe/camera_utils/camera_utils.js',
      'https://cdn.jsdelivr.net/npm/@mediapipe/drawing_utils/drawing_utils.js',
      'https://cdn.jsdelivr.net/npm/@mediapipe/pose/pose.js',
    ];
    let loaded = 0;
    srcs.forEach(src => {
      const s = document.createElement('script');
      s.src = src; s.crossOrigin = 'anonymous';
      s.onload = () => { if (++loaded === srcs.length) resolve(); };
      document.head.appendChild(s);
    });
  });
```

## 5. DB 마이그레이션 SQL



```sql
-- 결제 시스템
ALTER TABLE courses ADD COLUMN IF NOT EXISTS price INT DEFAULT 0;

CREATE TABLE IF NOT EXISTS payments (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  course_id INT NOT NULL,
  amount INT NOT NULL,
  status ENUM('pending','completed','refunded') DEFAULT 'completed',
  payment_method VARCHAR(50),
  card_last4 CHAR(4),
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS settlements (
  id INT AUTO_INCREMENT PRIMARY KEY,
  instructor_id INT NOT NULL,
  course_id INT NOT NULL,
  period VARCHAR(7) NOT NULL,
  revenue INT DEFAULT 0,
  payout_rate DECIMAL(3,2) DEFAULT 0.70,
  payout_amount INT DEFAULT 0,
  status ENUM('pending','paid') DEFAULT 'pending',
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (instructor_id) REFERENCES users(id),
  FOREIGN KEY (course_id) REFERENCES courses(id)
);

CREATE TABLE IF NOT EXISTS notifications (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  type VARCHAR(50) NOT NULL,
  title VARCHAR(200) NOT NULL,
  message TEXT,
  is_read TINYINT(1) DEFAULT 0,
  related_id INT,
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS activity_logs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT,
  action VARCHAR(100) NOT NULL,
  detail TEXT,
  ip_address VARCHAR(45),
  created_at DATETIME DEFAULT NOW()
);
```
