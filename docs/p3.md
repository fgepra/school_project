# P3 개발 진행 보고서 — 운영형 / 외부연동 + 관측성

> 작성일: 2026-06-12  
> 브랜치: `claude/distracted-northcutt`  
> 범위: P2-2 계획 기능 구현 + P3 신규 기능 (보안/이메일/디스코드/인덱스)

---

## 개요

5~6주차(P2-2)에서 설계만 했던 운영 기능(결제, 알림, 정산, 모니터링, 스케줄러, 로그)을 실제 코드로 완성하고,  
P3 단계에서 보안 강화(helmet/rate-limit/express-validator), 외부 서비스 연동(이메일/디스코드 웹훅),  
DB 인덱스 최적화, 이메일 구독 시스템까지 추가 구현했다.

## 1. DB 스키마

### 신규 테이블 6개

#### PAYMENTS — 결제 내역
```sql
CREATE TABLE IF NOT EXISTS payments (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  course_id INT NOT NULL,
  amount INT NOT NULL,
  status ENUM('pending','completed','refunded') DEFAULT 'completed',
  payment_method VARCHAR(50),
  card_last4 CHAR(4),
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE
);
```

#### SETTLEMENTS — 강사 정산
```sql
CREATE TABLE IF NOT EXISTS settlements (
  id INT AUTO_INCREMENT PRIMARY KEY,
  instructor_id INT NOT NULL,
  course_id INT NOT NULL,
  period VARCHAR(7) NOT NULL,
  revenue INT DEFAULT 0,
  payout_rate DECIMAL(3,2) DEFAULT 0.70,
  payout_amount INT DEFAULT 0,
  status ENUM('pending','paid') DEFAULT 'pending',
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (instructor_id) REFERENCES users(id),
  FOREIGN KEY (course_id) REFERENCES courses(id),
  UNIQUE KEY uq_settlement (instructor_id, course_id, period)
);
```

#### NOTIFICATIONS — 알림
```sql
CREATE TABLE IF NOT EXISTS notifications (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  type VARCHAR(50) NOT NULL,
  title VARCHAR(200) NOT NULL,
  message TEXT,
  is_read TINYINT(1) DEFAULT 0,
  related_id INT,
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

#### ACTIVITY_LOGS — 서버 활동 로그
```sql
CREATE TABLE IF NOT EXISTS activity_logs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT,
  action VARCHAR(100) NOT NULL,
  detail TEXT,
  ip_address VARCHAR(45),
  created_at DATETIME DEFAULT NOW()
);
```

#### SUBSCRIPTIONS — 이메일 구독 (P3 신규)
```sql
CREATE TABLE IF NOT EXISTS subscriptions (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  type ENUM('weekly_digest','course_update','promotion') DEFAULT 'weekly_digest',
  is_active TINYINT(1) DEFAULT 1,
  created_at DATETIME DEFAULT NOW(),
  UNIQUE KEY uq_user_type (user_id, type),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

#### COURSES 컬럼 추가
```sql
ALTER TABLE courses ADD COLUMN IF NOT EXISTS price INT DEFAULT 0;
```
- `price = 0` 이면 무료 강의.

### 성능 인덱스 (P3 신규)
```sql
CREATE INDEX IF NOT EXISTS idx_progress_user ON progress(user_id);
CREATE INDEX IF NOT EXISTS idx_progress_lecture ON progress(lecture_id);
CREATE INDEX IF NOT EXISTS idx_payments_user ON payments(user_id);
CREATE INDEX IF NOT EXISTS idx_payments_course ON payments(course_id);
CREATE INDEX IF NOT EXISTS idx_payments_status ON payments(status);
CREATE INDEX IF NOT EXISTS idx_notifications_user ON notifications(user_id, is_read);
CREATE INDEX IF NOT EXISTS idx_notifications_created ON notifications(created_at);
CREATE INDEX IF NOT EXISTS idx_lectures_course ON lectures(course_id, order_num);
CREATE INDEX IF NOT EXISTS idx_activity_logs_created ON activity_logs(created_at);
CREATE INDEX IF NOT EXISTS idx_activity_logs_user ON activity_logs(user_id);
```

## 2. 백엔드 구현

### 2-1. 보안 미들웨어 (P3 신규)

#### helmet.js — HTTP 보안 헤더
```js
app.use(helmet({ contentSecurityPolicy: false, crossOriginEmbedderPolicy: false }));
```
- `X-Content-Type-Options: nosniff` (MIME 스니핑 방어)
- `X-Frame-Options: DENY` (클릭재킹 방어)
- `Strict-Transport-Security` (HTTPS 강제)

#### express-rate-limit — 브루트포스 방지
```js
const loginLimiter = rateLimit({ windowMs: 15 * 60 * 1000, max: 20 });
app.use('/api/auth/login', loginLimiter);
```

#### express-validator — 입력값 검증 (SQL Injection / XSS 방어)
`backend/middleware/validationMiddleware.js`

| 검증 규칙 | 적용 경로 |
|-----------|----------|
| `registerRules` | POST /api/auth/register |
| `loginRules` | POST /api/auth/login |
| `commentRules` | POST /api/comments |
| `paymentRules` | POST /api/payments |

## 3. 프론트엔드 구현

### 3-1. Navbar 알림 벨 (`/components/layout/Navbar.tsx`)
- 30초마다 미읽음 수 폴링 → 빨간 뱃지 표시 (99+)
- 벨 클릭 시 드롭다운 — 최근 10개 알림, 클릭 시 읽음 처리
- [모두 읽음] 버튼, [전체 알림 보기] 링크
- 관리자 메뉴에 정산/로그/모니터링 추가
- 강사 메뉴에 정산 추가
- 학생 메뉴에 결제 내역 추가

### 3-2. 결제 페이지 (`/payment/[courseId]`)
3단계 플로우:
- **Step 1** — 카드사 선택(6개), 카드번호 자동 포맷(`0000-0000-0000-0000`), 유효기간, CVC, 할부
- **Step 2** — 입력 정보 요약 확인, 카드번호 마스킹
- **Step 3** — 1.5초 처리 스피너 → 완료 후 `/payment/complete` 이동

### 3-3. 결제 완료 페이지 (`/payment/complete`)
- 구매 강의명, 금액, 결제 수단, 환불 가능 기한(+7일) 표시
- [강의 목록] [결제 내역 보기] 버튼

### 3-4. 결제 내역 페이지 (`/payment/history`)
- 통계 카드: 총 결제 건수, 총 결제 금액, 총 환불 금액
- 결제 목록 테이블 + 완료 건에 [환불] 버튼

### 3-5. 알림 전체 목록 (`/notifications`)
- 타입별 아이콘: `payment → 💳`, `course_update → 📚`, `progress → 🏃`, `system → 🔔`
- 미읽음 = 파란 점 + 굵은 폰트 + 보라 배경
- [전체 읽음], 개별 [✕] 삭제 버튼

### 3-6. 강사 정산 페이지 (`/instructor/settlement`)
- 통계 카드: 총 매출, 총 정산액(70%), 정산 대기 건수
- 월별 그룹핑 테이블 + 정산 상세 목록

### 3-7. 관리자 정산 페이지 (`/admin/settlement`)
- 통계 카드: 총 결제 건수, 총 매출, 총 정산 예정액
- 기간 필터(일별/주별/월별) + 막대 차트
- 정산 수동 생성 (YYYY-MM 입력)
- CSV 다운로드 링크

### 3-8. 관리자 로그 페이지 (`/admin/logs`)
- action 드롭다운 필터
- 로그 테이블 (시각, 유저 ID, IP, 액션, 상세)
- [30일+ 로그 삭제] 버튼

### 3-9. 관리자 모니터링 페이지 (`/admin/monitor`)
- 헬스체크 카드: 서버 상태, DB 연결 (녹색/빨간 테두리)
- 서버 정보: 업타임, Node.js 버전, RSS/힙 메모리
- API 통계 4개: 총 요청수, 에러수, 에러율, 평균 응답시간
- 최근 요청 테이블 (메서드 색상, 상태코드 색상, 500ms+ 경고색)
- **15초 자동 갱신** + [새로고침] 버튼

### 3-10. 이메일 구독 설정 (`/settings/subscriptions`) — P3 신규
- 토글 스위치 UI로 3가지 구독 타입 개별 설정
- `weekly_digest` / `course_update` / `promotion`

### 3-11. 강의 상세 페이지 결제 연동 (`/courses/[id]`)
- 유료 강의(`price > 0`): [₩N,000 구매하기] 버튼
- 미로그인: [로그인 후 구매] 버튼
- 구매 완료: 녹색 [구매 완료] 뱃지
- 페이지 진입 시 구매 여부 자동 확인
