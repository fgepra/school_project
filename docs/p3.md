# P3 개발 설계 문서 — 운영형 / 외부연동 + 관측성

> 작성일: 2026-06-11  
> 브랜치: `claude/distracted-northcutt`  
> 담당: 보안 강화, 이메일/디스코드 구독, Webhook, 결제 PG 연동, DB 최적화, 프론트엔드 성능, 배포 최적화

---

## 개요

P3는 서비스를 **실제 운영 가능한 수준**으로 완성하는 단계다.  
P2-2에서 계획했던 기본 운영 기능(결제 시뮬레이션, 알림, 스케줄러, 모니터링, 정산)을 코드로 구현하고,  
외부 서비스 연동(이메일, 디스코드), 보안 강화(helmet/XSS/SQL Injection 방어),  
DB 인덱스 최적화, 프론트엔드 렌더링 성능 개선, 실제 PG 결제 연동까지 확장한다.

## 구현 범위

### 1. 보안 강화 (Security)

#### 1-1. helmet.js — HTTP 보안 헤더
```js
app.use(helmet());
```
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Content-Security-Policy`
- `Strict-Transport-Security`

#### 1-2. express-rate-limit — 브루트포스 방지
```js
const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,  // 15분
  max: 10,                    // 최대 10회
  message: { message: '요청이 너무 많습니다. 15분 후 다시 시도해주세요.' }
});
app.use('/api/auth/login', loginLimiter);
```

#### 1-3. express-validator — 입력값 검증 (SQL Injection / XSS 방어)
```js
const { body, validationResult } = require('express-validator');

// 회원가입 검증
const registerValidation = [
  body('email').isEmail().normalizeEmail(),
  body('password').isLength({ min: 6 }).trim(),
  body('name').notEmpty().trim().escape(),
];
```

### 3. 결제 시스템 (Payment)

#### DB 테이블
```sql
ALTER TABLE courses ADD COLUMN IF NOT EXISTS price INT DEFAULT 0;

CREATE TABLE IF NOT EXISTS payments (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  course_id INT NOT NULL,
  amount INT NOT NULL,
  status ENUM('pending','completed','refunded') DEFAULT 'completed',
  payment_method VARCHAR(50),
  card_last4 CHAR(4),
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE
);
```

#### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/payments` | 결제 처리 (시뮬레이션) |
| GET | `/api/payments/my` | 내 결제 내역 |
| GET | `/api/payments/check/:courseId` | 구매 여부 확인 |
| POST | `/api/payments/:id/refund` | 환불 |
| GET | `/api/payments/all` | 전체 결제 (관리자) |

### 4. 알림 시스템 (Notifications)

#### DB 테이블
```sql
CREATE TABLE IF NOT EXISTS notifications (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  type VARCHAR(50) NOT NULL,
  title VARCHAR(200) NOT NULL,
  message TEXT,
  is_read TINYINT(1) DEFAULT 0,
  related_id INT,
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 5. 정산 시스템 (Settlement)

#### DB 테이블
```sql
CREATE TABLE IF NOT EXISTS settlements (
  id INT AUTO_INCREMENT PRIMARY KEY,
  instructor_id INT NOT NULL,
  course_id INT NOT NULL,
  period VARCHAR(7) NOT NULL,
  revenue INT DEFAULT 0,
  payout_rate DECIMAL(3,2) DEFAULT 0.70,

  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (instructor_id) REFERENCES users(id),
  FOREIGN KEY (course_id) REFERENCES courses(id)
);
```

### 7. 스케줄러 (node-cron)

```js
// 매일 새벽 2시 — 30일 이상 오래된 로그 삭제
cron.schedule('0 2 * * *', () => { ... });

// 매월 1일 자정 — 전월 정산 자동 생성

// 매주 월요일 오전 9시 — 미완료 진도 독려 알림
cron.schedule('0 9 * * 1', () => { ... });

// 매일 오전 8시 — 신규 공지 이메일 발송 (P3 신규)
cron.schedule('0 8 * * *', () => { sendDailyDigest(); });
```

### 8. 이메일 구독 (Nodemailer) — P3 신규

#### 이메일 발송 유틸
```js
// backend/utils/mailer.js
const transporter = nodemailer.createTransport({
  service: 'gmail',    // 또는 SES
  auth: { user: process.env.MAIL_USER, pass: process.env.MAIL_PASS }
});

const sendMail = async ({ to, subject, html }) => {
  await transporter.sendMail({ from: '"HOMEFIT" <noreply@homefit.app>', to, subject, html });
};
```

#### 이메일 구독 테이블
```sql
CREATE TABLE IF NOT EXISTS subscriptions (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  type ENUM('weekly_digest','course_update','promotion') DEFAULT 'weekly_digest',
  is_active TINYINT(1) DEFAULT 1,
  created_at DATETIME DEFAULT NOW(),
  UNIQUE KEY uq_user_type (user_id, type),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 9. 디스코드 웹훅 — P3 신규

```js
// backend/utils/discord.js
const sendDiscordAlert = async (content) => {
  if (!process.env.DISCORD_WEBHOOK_URL) return;
  await fetch(process.env.DISCORD_WEBHOOK_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ content }),
  });
};

// 사용 예시
sendDiscordAlert(`🎓 신규 결제: ${user.name} → ${course.title} (₩${amount.toLocaleString()})`);
sendDiscordAlert(`⚠️ 서버 에러 급증: 에러율 ${errorRate}%`);
```

### 11. 프론트엔드 성능 최적화 — P3 신규

- `React.memo` — 정적 카드 컴포넌트 불필요한 재렌더링 방지
- `useMemo` — 완료율 계산, 필터링 결과 캐싱
- `useCallback` — 이벤트 핸들러 안정화
- `dynamic import` — 무거운 컴포넌트(모션 캡처 패널) 코드 스플리팅
- `Suspense` + `loading.tsx` — 스트리밍 렌더링
