## 개요

1~2주차에 구현된 학생 전용 기능(회원가입/로그인, 강의 조회, 시청, 진도 관리)을 기반으로,  
3주차에는 **관리자 / 강사 / 학생** 세 가지 역할을 분리하고 각 역할에 맞는 UI와 기능을 추가했다.  
또한 학생이 댓글을 작성하고 강사가 답글을 다는 상호작용 기능도 함께 구현했다.

## 1. DB 스키마 확장

### 변경 파일: `docs/p1.md`

기존 4개 테이블(`USERS`, `COURSES`, `LECTURES`, `PROGRESS`)에서 다음을 추가/변경했다.

#### USERS 테이블 — `role` 컬럼 추가
```sql
ALTER TABLE users
  ADD COLUMN role ENUM('student', 'instructor', 'admin') NOT NULL DEFAULT 'student';
```
- 기존 회원은 모두 `student`로 초기화된다.
- 회원가입 시 `student` 또는 `instructor`만 선택 가능하며, `admin`은 DB에서 직접 지정한다.

#### COURSES 테이블 — `instructor_id` 컬럼 추가
```sql
ALTER TABLE courses
  ADD COLUMN instructor_id INT NULL,
  ADD FOREIGN KEY (instructor_id) REFERENCES users(id) ON DELETE SET NULL;
```
- 강의를 생성한 강사의 ID를 저장한다.
- 강사가 탈퇴해도 강의는 유지되도록 `ON DELETE SET NULL` 설정.

#### LECTURES 테이블 — `order_num` 컬럼 추가
```sql
ALTER TABLE lectures
  ADD COLUMN order_num INT NOT NULL DEFAULT 0;
```
- 강의 영상의 재생 순서를 지정한다.

#### COMMENTS 테이블 — 신규 생성
```sql
CREATE TABLE comments (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  user_id    INT NOT NULL,
  lecture_id INT NOT NULL,
  content    TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id)    REFERENCES users(id)    ON DELETE CASCADE,
  FOREIGN KEY (lecture_id) REFERENCES lectures(id) ON DELETE CASCADE
);
```
- 학생이 강의 영상에 남기는 질문/의견 댓글.
- 강의 또는 유저 삭제 시 댓글도 함께 삭제된다.

#### REPLIES 테이블 — 신규 생성
```sql
CREATE TABLE replies (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  comment_id INT NOT NULL,
  user_id    INT NOT NULL,
  content    TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (comment_id) REFERENCES comments(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id)    REFERENCES users(id)    ON DELETE CASCADE
);
```
- 강사/관리자가 댓글에 다는 답글.
- 댓글 삭제 시 답글도 함께 삭제된다.

## 2. 백엔드

### 2-1. 인증 (`backend/controllers/authController.js`)

**변경 이유**: 기존에는 `role` 없이 JWT를 발급했기 때문에 역할 기반 제어가 불가능했다.

**변경 내용**:
- `register`: `role` 파라미터를 받아 저장. `student` / `instructor`만 허용하고, 그 외 값은 강제로 `student`로 처리.
- `login`: 응답 user 객체에 `role` 포함, JWT payload에도 `{ id, role }` 포함.
- JWT 만료 시간을 `1h` → `7d`로 변경 (자동 로그아웃 빈도 감소).

```js
// JWT에 role 포함
const token = jwt.sign(
  { id: user.id, role: user.role },
  process.env.JWT_SECRET,
  { expiresIn: "7d" }
);
```

### 2-2. 역할 미들웨어 (`backend/middleware/roleMiddleware.js`) — 신규

**역할**: `authMiddleware`로 JWT를 검증한 뒤, 추가로 역할을 확인하는 미들웨어.

```js
const requireRole = (...roles) => (req, res, next) => {
  if (!roles.includes(req.user.role)) {
    return res.status(403).json({ message: "접근 권한이 없습니다." });
  }
  next();
};

const requireAdmin      = requireRole('admin');
const requireInstructor = requireRole('instructor', 'admin'); // 관리자도 강사 권한 포함
```

- `requireAdmin`: 관리자만 허용
- `requireInstructor`: 강사 또는 관리자 허용

### 2-3. 관리자 API (`backend/controllers/adminController.js`, `backend/routes/admin.js`) — 신규

**주요 로직**:

### 2-4. 강사 API (`backend/controllers/instructorController.js`, `backend/routes/instructor.js`) — 신규

모든 라우트에 `authMiddleware + requireInstructor` 적용.

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/instructor/courses` | 내 강의 목록 (관리자는 전체) |
| POST | `/api/instructor/courses` | 강의 생성 |
| PUT | `/api/instructor/courses/:courseId` | 강의 수정 |
| DELETE | `/api/instructor/courses/:courseId` | 강의 삭제 |
| GET | `/api/instructor/courses/:courseId/lectures` | 강의 영상 목록 |
| POST | `/api/instructor/courses/:courseId/lectures` | 강의 영상 추가 |
| PUT | `/api/instructor/lectures/:lectureId` | 강의 영상 수정 |
| DELETE | `/api/instructor/lectures/:lectureId` | 강의 영상 삭제 |

**주요 로직**:
- 강의 수정/삭제 전 `instructor_id = req.user.id` 조건으로 본인 강의 여부를 검증.
- 관리자(`role === 'admin'`)는 조건 없이 모든 강의에 접근 가능.

```js
// 본인 강의인지 확인 (관리자 예외)
const checkSql = isAdmin
  ? "SELECT id FROM courses WHERE id = ?"
  : "SELECT id FROM courses WHERE id = ? AND instructor_id = ?";
```

### 2-5. 댓글 API (`backend/controllers/commentController.js`, `backend/routes/comments.js`) — 신규

| 메서드 | 경로 | 권한 | 설명 |
|--------|------|------|------|
| GET | `/api/comments/lecture/:lectureId` | 인증 유저 | 댓글 + 답글 목록 |
| POST | `/api/comments/lecture/:lectureId` | 인증 유저 | 댓글 작성 |
| DELETE | `/api/comments/:commentId` | 본인 or 관리자 | 댓글 삭제 |
| POST | `/api/comments/:commentId/replies` | 강사 / 관리자 | 답글 작성 |
| DELETE | `/api/comments/replies/:replyId` | 본인 or 관리자 | 답글 삭제 |

**댓글 목록 조회 로직**:
1. 해당 강의의 댓글을 전부 조회 (유저 이름, 역할 JOIN).
2. 댓글 ID 목록으로 답글을 한 번에 조회 (`IN (?)` 쿼리).
3. JS에서 댓글 객체에 `replies` 배열을 매핑하여 중첩 구조로 응답.

```js
const result = comments.map(comment => ({
  ...comment,
  replies: replies.filter(r => r.comment_id === comment.id),
}));
```

#### 영상 관리 (`/instructor/courses/[courseId]/lectures`)

- 영상 추가 폼: 제목, 순서, URL, 재생시간(초) 입력.
- 각 영상 삭제 버튼 (확인 다이얼로그 포함).


## 1. DB 스키마 추가

### WORKOUT_LOGS 테이블

```sql
CREATE TABLE IF NOT EXISTS workout_logs (
  id         INT AUTO_INCREMENT PRIMARY KEY,
  user_id    INT NOT NULL,
  date       DATE NOT NULL,
  duration   INT NOT NULL DEFAULT 0,   -- 운동 시간(분)
  calories   FLOAT NOT NULL DEFAULT 0, -- 소모 칼로리
  met_value  FLOAT NOT NULL DEFAULT 0, -- 운동 강도 지표
  memo       TEXT,
  created_at DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

- **MET × 체중(kg) × 시간(h) = 소모 칼로리** 공식을 서버에서 계산하지 않고 프론트에서 계산 후 저장.
- `date` 컬럼으로 날짜별 조회/집계 가능.

### BOOKMARKS 테이블

```sql
CREATE TABLE IF NOT EXISTS bookmarks (
  id        INT AUTO_INCREMENT PRIMARY KEY,
  user_id   INT NOT NULL,
  course_id INT NOT NULL,
  created_at DATETIME DEFAULT NOW(),
  UNIQUE KEY uq_user_course (user_id, course_id),
  FOREIGN KEY (user_id)   REFERENCES users(id)    ON DELETE CASCADE,
  FOREIGN KEY (course_id) REFERENCES courses(id)  ON DELETE CASCADE
);
```

- `UNIQUE KEY`로 중복 북마크 방지. 토글 방식으로 INSERT/DELETE 처리.

### 3-2. 모션 캡처 페이지 (`/motion`)

MediaPipe Pose를 활용해 웹캠 영상에서 관절 좌표를 추출하고 자세를 분석한다.

#### MediaPipe CDN 로딩 방식

Turbopack이 CJS 형식의 `@mediapipe/pose` npm 패키지를 번들링하지 못하는 문제가 있어,  
CDN에서 스크립트를 동적으로 로드하고 `window` 전역 객체로 접근하는 방식을 채택했다.

```ts
const loadMediaPipe = (): Promise<void> => {
  return new Promise((resolve) => {
    if (window.Pose) { resolve(); return; }
    const scripts = [
      'https://cdn.jsdelivr.net/npm/@mediapipe/camera_utils/camera_utils.js',
      'https://cdn.jsdelivr.net/npm/@mediapipe/drawing_utils/drawing_utils.js',
      'https://cdn.jsdelivr.net/npm/@mediapipe/pose/pose.js',
    ];
    let loaded = 0;
    scripts.forEach((src) => {
      const s = document.createElement('script');
      s.src = src;
      s.onload = () => { if (++loaded === scripts.length) resolve(); };
      document.head.appendChild(s);
    });
  });
};
```

#### 지원 운동 및 자세 분석

| 운동 | 분석 항목 | 카메라 방향 안내 |
|------|-----------|-----------------|
| 스쿼트 | 무릎 각도 (170° 이상 = STANDING, 90° 미만 = DOWN) | 옆면 촬영 권장 |
| 푸시업 | 팔꿈치 각도 (160° 이상 = UP, 70° 미만 = DOWN) | 옆면 촬영 권장 |
| 플랭크 | 어깨-고관절-발목 직선도 (각도 ±15° 이내 = GOOD) | 옆면 촬영 권장 |

**카메라 안내 배너**: 스쿼트/푸시업/플랭크 선택 시 노란색 경고 배너로 측면 촬영 안내 표시.

**분석 로직** — 3점 각도 계산:
```ts
const calcAngle = (a: Point, b: Point, c: Point): number => {
  const ab = Math.atan2(a.y - b.y, a.x - b.x);
  const cb = Math.atan2(c.y - b.y, c.x - b.x);
  return Math.abs((ab - cb) * 180 / Math.PI);
};
```

**반복 횟수 카운팅**: `UP → DOWN → UP` 전환 감지로 1회 카운트.  
Canvas에 랜드마크 + 연결선을 실시간 오버레이로 렌더링.

```ts
  dragItem.current = index;

const handleDrop = (e: DragEvent, dropIndex: number) => {
  const reordered = [...lectures];
  const [moved] = reordered.splice(dragItem.current!, 1);
  reordered.splice(dropIndex, 0, moved);
  setLectures(reordered.map((l, i) => ({ ...l, order_num: i + 1 })));
  instructorApi.reorderLectures(courseId, reordered.map((l, i) => ({ id: l.id, order_num: i + 1 })));
};
```
